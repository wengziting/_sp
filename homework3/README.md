這台「虛擬電腦」是由三個角色分工合作完成的：

## 1. 專案核心邏輯總結
記憶體 (Memory)：就像一個裝有 256 個格子的櫃子，每個格子只能放一個 0 到 255 之間的數字。它負責存放你的「程式碼」和「資料」。  

CPU (中央處理器)：這是電腦的大腦。它有一個「程式計數器 (PC)」來記住現在讀到櫃子的第幾個格子，還有幾個「暫存器 (Registers)」用來暫時存放運算中的數字（就像大腦的短期記憶）。  

指令集 (Instruction Set)：這是你跟電腦溝通的語言。例如你規定數字 0x01 代表「搬東西 (LOAD)」，數字 0x02 代表「加法 (ADD)」。

## 2. 電腦運作流程 (Fetch-Decode-Execute)
當你執行 node src/main.js 時，後台會發生以下循環：

擷取 (Fetch)：CPU 根據 PC 的位置，從記憶體櫃子裡拿出一個數字。  

解碼 (Decode)：CPU 查看這個數字，對照開關電路（switch-case），發現它是 0x02，知道這是「加法指令」。  

執行 (Execute)：CPU 真的去把兩個暫存器的數字加起來，存回結果，然後把 PC 往後移，準備讀下一個指令。
