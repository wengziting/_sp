## 第一階段：詞法分析 (Lexer) —— 把文字變成「單字」
電腦看不懂 func main() { ... } 這一整串字串。詞法分析器（next_tk）就像是一個分類器，它會逐字掃描，將字串切碎並貼上標籤（Token）。

動作：遇到 123 標記為 TK_NUM；遇到 while 標記為關鍵字。

過濾：它會自動跳過空格和註解，讓後面的步驟只需要處理「有意義的資訊」。

## 第二階段：語法分析 (Parser) —— 理解「句型結構」
有了單字後，需要檢查它們是否符合語法規則。這部分使用遞歸下降解析（Recursive Descent Parsing），它模仿了人類理解句子的方式。

優先權處理：透過 arith -> term -> factor 的層級呼叫，確保電腦會先算「乘除」再算「加減」。

控制流：當解析到 if 或 while 時，它會理解這是一個「跳轉邏輯」，並預留位置紀錄「如果條件不成立要跳到哪裡」。

嚴謹性：透過 expect() 函數，如果語法少了一個 ( 或 ;，它會立刻報錯，而不是產生錯誤的指令。

## 第三階段：中間代碼生成 (Intermediate Code) —— 轉換成「說明書」
這步是編譯器的精華。它不直接執行，而是把複雜的語法簡化成一種格式：四元組 (Quadruples)。
格式為：(操作, 參數1, 參數2, 結果)。

例如 a = (b + 5) * 2; 會被拆解成：

ADD b 5 t1 (把 b+5 存入臨時變數 t1)

MUL t1 2 t2 (把 t1*2 存入臨時變數 t2)

STORE t2 - a (把 t2 存入 a)

這樣做的目的是讓最後的執行階段（VM）變得非常簡單，只需要像搬運工一樣按表操課。

## 第四階段：虛擬機器 (Virtual Machine) —— 模擬「大腦執行」
這部分（vm_execute）模擬了一台虛擬電腦，它有自己的「記憶體」和「執行邏輯」。

棧幀 (Stack Frame)：這是最重要的地方。每當你呼叫一個函數，VM 就會在記憶體堆疊（Stack）中推入一個新的空間。

這塊空間存放該函數的區域變數和參數。

當函數結束（RET_VAL），這個空間會被銷毀，並跳回原本的地方繼續執行。

變數查找：get_var 與 set_var 負責在目前的記憶體空間尋找變數的值。如果是數字（如 10），它會直接處理；如果是變數名（如 a），它會去記憶體表格裡查。
